    How it works

    Adaptive Bayesian AI (default, strongest). It maintains thousands of complete legal fleet hypotheses and filters them after every shot. Unlike the earlier version, the game reports the exact cells and length whenever a ship sinks, so the AI removes that ship without guessing. The surviving hypotheses are rejuvenated with constrained Gibbs moves, and shots are ranked using occupancy, partially-hit-ship continuation, sink probability, and a shallow two-shot lookahead.

    Learning across games. Completed games are stored only in this browser. The attacker uses previous player layouts as an empirical prior, while the placement system learns which cells the player tends to fire at early. The memory is capped at 40 games and can be erased with Reset AI Memory. No data leaves the browser.

    Probability AI. A fast deterministic approximation that enumerates every legal placement for each remaining ship length. It now also receives exact sunk-ship information, making it a substantially stronger baseline.

    Ship placement. The enemy chooses from a diverse pool of offline-optimized layouts. It then favors layouts that survived longest against the player's previous firing orders while keeping a randomized mixed strategy, so repeated games do not expose one fixed pattern.

    Static, client-side JS — no backend. See ai.js for the AI implementation.
